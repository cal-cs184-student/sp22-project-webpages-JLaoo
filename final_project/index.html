Extending CPU Cloth Simulation Using GPU Shaders

Team Members: Jonathan Luo, Jacob Hsiung, William Chen, Zoltan Williamson



Summary

In this project, we hope to use a more powerful shader API in order to speed up rendering times for a cloth simulation and/or other simulations. Along with these improved rendering times, we could also implement additional features such as more accurate self collision.

Project Description

In project 4, we implemented our shader in GLSL, but we could implement the shader in OpenGL or another powerful shader library to attain faster rendering times, which would allow us to implement new features without needing to sacrifice framerate. This is an important issue to solve as rendering times are a major bottleneck when running real time simulations. By reducing rendering time, we will not only be able to increase framerate, but also have the chance to implement more advanced features without sacrificing too much framerate. We believe one of the main challenges of this project is learning how to incorporate OpenGL properly, as it is an API that we are not familiar with. If we decide to implement more advanced physics, it will also be a challenge to translate that physics into code.

Goals and Deliverables

In this project, we’ll attempt to speedup the rendering times of ClothSim from project 4 by utilizing the GPU via the OpenGL API. In order to demonstrate our implementation, we plan on running simulations at a high speed at a frame rate that the current project 4 implementation would be unable to match. We also plan on including tables/graphs of rendering times and/or framerates to serve as metrics to determine the speedup we’re able to obtain. Through this analysis, we hope to answer the question of exactly how much of a speedup OpenGL provides in ClothSim and perhaps other various simulations.

Plan to deliver: A sped up version of ClothSim utilizing the OpenGL API along with sped up simple simulations (such as the ones currently in project 4). Graphs and/or tables of metrics such as framerates and rendering times will also be included. We plan on adding on to the current project 4 implementation. In the event that we deem it infeasible to work with the existing project 4 codebase (ie it may be too difficult to transform the current codebase into one that can work with OpenGL), we plan on writing a cloth simulation application from scratch and comparing our new implementation’s metrics with the metrics from project 4.

Hope to deliver: If we have time, we would like to also create more complicated simulations and compare metrics. We would also like to implement new features such as additional textures, materials, or more accurate physics (eg self collision) if we have time.

Schedule

  • Week 1: Determine whether project 4 codebase can be used for incorporating OpenGL for GPU speedup. Read up on OpenGL tutorials/APIs to get a better understanding of how the API works. Play around with the OpenGL library.
  • Week 2: Start incorporating OpenGL into old codebase, or start from scratch with OpenGL (depending on our research from week 1), start adding additional animations to demonstrate speedup.
  • Week 3: Finish working on OpenGL incorporation. If have time, add new features now possible due to speedup.
  • Week 4: Finish up any coding tasks, final debugging/testing, presentation preparation (slides, generating final simulations/graphs/tables, etc.)
